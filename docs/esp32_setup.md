# ESP32 Setup Guide\n## Portable Solar Power Generator\n\n### Hardware Requirements\n- ESP32 Development Board\n- Relay Module (4-channel)\n- Voltage Divider Circuits\n- ACS712 Current Sensor\n- Jumper Wires\n- Breadboard/PCB\n\n### Pin Connections\n\n#### Analog Inputs\n- **A0** - Solar Voltage (with voltage divider)\n- **A3** - Battery Voltage (with voltage divider) \n- **A6** - Current Sensor (ACS712)\n\n#### Digital Outputs - Relay Control\n- **Pin 2** - Light Relay\n- **Pin 4** - Fan Relay\n- **Pin 5** - Mobile Charger Relay\n- **Pin 18** - System Main Relay\n\n#### Status LEDs\n- **Pin 19** - Power LED (Green)\n- **Pin 21** - Charging LED (Blue)\n- **Pin 22** - Low Battery LED (Red)\n\n### Software Setup\n\n#### 1. Install Arduino IDE\n```bash\n# Download from: https://www.arduino.cc/en/software\n```\n\n#### 2. Install ESP32 Board Package\n1. Open Arduino IDE\n2. Go to File → Preferences\n3. Add this URL to Additional Board Manager URLs:\n   ```\n   https://dl.espressif.com/dl/package_esp32_index.json\n   ```\n4. Go to Tools → Board → Board Manager\n5. Search \"ESP32\" and install\n\n#### 3. Install Required Libraries\n```cpp\n// Install these libraries via Library Manager:\n- ArduinoJson (by Benoit Blanchon)\n- WiFi (ESP32 built-in)\n- WebServer (ESP32 built-in)\n- EEPROM (ESP32 built-in)\n```\n\n#### 4. Upload Code\n1. Open `esp32_controller.ino`\n2. Select Board: \"ESP32 Dev Module\"\n3. Select correct COM Port\n4. Click Upload\n\n### WiFi Configuration\n\n#### Default Settings\n- **SSID:** Solar_Generator_AP\n- **Password:** solar123\n- **IP Address:** 192.168.4.1\n\n#### Web Interface Access\n1. Connect to WiFi: \"Solar_Generator_AP\"\n2. Open browser: http://192.168.4.1\n3. Control devices via web interface\n\n### Calibration\n\n#### Voltage Sensors\n```cpp\n// Adjust these calibration factors in code:\nsolarVoltage = (solarRaw * 3.3 / 4095.0) * 7.58;  // For 0-25V range\nbatteryVoltage = (batteryRaw * 3.3 / 4095.0) * 4.55;  // For 0-15V range\n```\n\n#### Current Sensor (ACS712)\n```cpp\n// For ACS712-20A:\nchargingCurrent = (currentVoltage - 1.65) / 0.1;\n```\n\n### Safety Features\n\n#### Automatic Protection\n- **Low Voltage Cutoff:** < 10.5V\n- **Load Disconnect:** < 15% battery\n- **Emergency Shutdown:** Manual via web/button\n- **State Persistence:** EEPROM storage\n\n### Troubleshooting\n\n#### Common Issues\n1. **WiFi not connecting:**\n   - Check ESP32 power supply\n   - Verify antenna connection\n\n2. **Sensors not reading:**\n   - Check voltage divider ratios\n   - Verify analog pin connections\n\n3. **Relays not switching:**\n   - Check relay module power (5V)\n   - Verify control pin connections\n\n#### Serial Monitor Output\n```\nSolar Power Generator ESP32 Starting...\nAP IP address: 192.168.4.1\nWiFi AP started: Solar_Generator_AP\nWeb server started on port 80\nSystem initialized successfully!\n\n=== System Status ===\nSolar Voltage: 16.45V\nBattery Voltage: 12.34V\nBattery Percentage: 87.6%\nCharging Current: 1.23A\nCharging: Yes\nLight: OFF\nFan: OFF\nCharger: OFF\n```\n\n### API Endpoints\n\n#### GET /api/status\nReturns current system status in JSON format\n\n#### POST /api/control\nControl devices:\n```json\n{\n  \"device\": \"light\",\n  \"action\": \"toggle\"\n}\n```\n\n#### POST /api/emergency\nExecute emergency shutdown\n\n### Integration with Python Code\n\nThe ESP32 works alongside the Python application:\n- **ESP32:** Hardware control and WiFi interface\n- **Python:** Data logging, mobile app, advanced features\n- **Communication:** HTTP API calls between systems\n\n### Future Enhancements\n- MQTT integration\n- Mobile app direct connection\n- Cloud data logging\n- OTA (Over-The-Air) updates
